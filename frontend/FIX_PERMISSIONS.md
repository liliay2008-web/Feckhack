# Решение проблемы "Permission denied" во фронтенде

## Возможные причины и решения

### 1. Порт 3000 занят или требует прав администратора

**Решение:**
```bash
# Проверьте, какой процесс использует порт 3000
# Windows:
netstat -ano | findstr :3000

# Linux/Mac:
lsof -i :3000

# Убейте процесс или используйте другой порт
```

**Или измените порт в vite.config.js:**
```javascript
server: {
  port: 3001, // или любой другой свободный порт
  // ...
}
```

### 2. Проблемы с правами доступа к файлам

**Windows:**
```powershell
# Запустите PowerShell от имени администратора
# Перейдите в директорию проекта
cd frontend

# Установите права на запись
icacls . /grant Users:F /T

# Или запустите терминал от имени администратора
```

**Linux/Mac:**
```bash
# Установите права на директорию проекта
chmod -R 755 frontend
chmod -R 755 frontend/node_modules

# Или используйте sudo (не рекомендуется для node_modules)
sudo chown -R $USER:$USER frontend
```

### 3. Очистка кэша и переустановка зависимостей

```bash
cd frontend

# Удалите node_modules и lock файлы
rm -rf node_modules
rm -f package-lock.json

# Очистите кэш npm
npm cache clean --force

# Переустановите зависимости
npm install
```

### 4. Запуск с правами администратора (Windows)

```powershell
# Запустите PowerShell от имени администратора
# Затем:
cd frontend
npm run dev
```

### 5. Использование другого порта

Если порт 3000 недоступен, можно запустить на другом порту:

```bash
# Установите переменную окружения
export PORT=3002  # Linux/Mac
set PORT=3002     # Windows CMD
$env:PORT=3002    # Windows PowerShell

# Или измените vite.config.js
```

### 6. Проверка файловой системы

Убедитесь, что:
- Проект не находится в защищенной директории (например, Program Files)
- У вас есть права на запись в директорию проекта
- Антивирус не блокирует доступ к файлам

### 7. Альтернативный способ запуска

```bash
# Используйте npx для запуска
npx vite --port 3000 --host 0.0.0.0

# Или с явным указанием конфига
npx vite --config vite.config.js
```

## Быстрое решение

1. **Закройте все процессы, использующие порт 3000**
2. **Очистите кэш:** `npm cache clean --force`
3. **Переустановите зависимости:** `rm -rf node_modules && npm install`
4. **Запустите с правами администратора** (если нужно)
5. **Используйте другой порт**, если проблема сохраняется

## Проверка конфигурации

Убедитесь, что в `vite.config.js` установлено:
```javascript
server: {
  port: 3000,
  host: '0.0.0.0',
  strictPort: false, // Это позволит использовать другой порт автоматически
}
```
